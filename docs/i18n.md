# Localization (i18n) API

**Doc type:** Java API · **Verified against 0.5.1**

Events for the internationalization/localization system.

## Overview

Implemented in `com.hypixel.hytale.server.core.modules.i18n.event` and provides:
- A startup event for registering plugin-supplied default translations
- Translation files keyed by namespace, supplied via a `TranslationMap`
- Translation keys consumed by player-facing messages and UI elements

## Architecture
```
Server startup
└── GenerateDefaultLanguageEvent
      └── putTranslationFile(namespace, TranslationMap)
            └── TranslationMap (key ──▶ translated string)
```

## Usage Example

```java
import com.hypixel.hytale.server.core.modules.i18n.event.GenerateDefaultLanguageEvent;

@Override
protected void setup() {
    // Register default translations
    getEventRegistry().register(GenerateDefaultLanguageEvent.class, event -> {
        TranslationMap translations = new TranslationMap();
        translations.put("my_plugin.greeting", "Hello, world!");
        translations.put("my_plugin.farewell", "Goodbye!");

        event.putTranslationFile("my_plugin", translations);
    });
}
```

> **See also:** [Player API - Message](player.md#message)

## Resolving a key to text (server-side)

`GenerateDefaultLanguageEvent` *registers* translations; `I18nModule` *reads* one back. You need this when something can only accept a plain `String`, not a `Message` — e.g. a [`CustomUIHud`](ui-api.md#customuihud) label, which disconnects the client if you hand it a `Message`. You also can't defer localization to the client there, so resolve the key server-side first.

**Package:** `com.hypixel.hytale.server.core.modules.i18n`

| Member | Return Type | Description |
|--------|-------------|-------------|
| `I18nModule.get()` | `I18nModule` | The module singleton |
| `getMessage(String language, String key)` | `String` | Translated text for `key` in `language` (`getMessages(language).get(key)`) |
| `getMessages(String language)` | `Map<String, String>` | All key→text pairs for a language |
| `I18nModule.DEFAULT_LANGUAGE` | `static final String` | Fallback language to pass when you have no per-client locale |

```java
import com.hypixel.hytale.server.core.modules.i18n.I18nModule;

// Resolve an entity's display-name key (a translation) to plain text
String name = I18nModule.get()
    .getMessage(I18nModule.DEFAULT_LANGUAGE, "npcRoles.Bear_Grizzly.name"); // → "Grizzly Bear"
```

Translation values for the built-in languages live in `Server/Languages/<lang>/server.lang` (e.g. `en-US`). NPC display-name keys look like `npcRoles.<Role>.name`; an entity's `DisplayNameComponent.getDisplayName()` is itself a translation `Message`, so read its [`getMessageId()`](player.md#getters) and pass that here.

## Notes

- Use this event to register plugin translations during server startup
- Translation keys should be namespaced with your plugin name to avoid conflicts
- Translations are used for player-facing messages and UI elements

---

## Gotchas & Errors

- **Symptom:** a translation entry is rejected at startup with an "invalid default translation message" log line → the entry had a null/empty key or value (the loader validates each `key=value` pair). Fix: ensure every `TranslationMap.put(key, value)` uses a non-empty, namespaced key and a non-null value.
- **Symptom:** a `Message.translation("…")` (or `%key` in a `.ui` file) renders as the raw key on the client → no translation was registered for that key. Fix: register it via `putTranslationFile` here at startup; keys are namespaced, so the key must match exactly (see [Usage Example](#usage-example)).
